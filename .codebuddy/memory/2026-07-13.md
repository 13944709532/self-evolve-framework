## 知识库重构：按语言/框架分目录

为长期可维护性，将原单文件 `project-gotchas.mdc` 拆分为按语言/框架分目录的知识库：

### 新目录结构 `template/rules/knowledge/`
- `README.md`：索引 + 归属约定 + 编号体系（F 强制规则 / P 多发问题，主线 F1–F13、P1–P15；专项可用 `F-rust-1` 子编号）
- `general.mdc`（alwaysApply: true）：跨语言通用（命名桥接、错误友好、受保护组件、token 样式、P1 同字段AND零行、P2 SQLite中文列名、P3 SQLite禁TRUNCATE、P15 文档漂移）
- `typescript.mdc`（按需）：TS 命名、strict/禁用any、注释语法、P7 依赖先装
- `vue.mdc`（按需）：组件命名、F2 v-model 禁双watch、F7 响应式、F8 弹窗懒加载、F12 DOM时序、P12 跨Win样式、P11 图表空数据
- `rust.mdc`（按需）：Rust 命名、F4 DB类型容错、F11 XLSX工厂、P4 MutexGuard、P8 invoke_handler单宏、P13 全局状态、P14 连接诊断
- `tauri.mdc`（按需）：IPC 通信、TS↔Rust 桥接、F9 Dialog选择、F10 落盘走Dialog+Rust、P5 双配置同步、P6 配置字段cargo check、P-tauri-1 绕过浏览器下载

### 关键设计决策
- **加载策略**：`general.mdc` always 激活；语言/框架专项 `alwaysApply: false`，靠 description 语义检索在对应上下文按需加载，避免一次性注入全部 context 导致膨胀。
- **归属原则**：跨语言通用放 general；仅语言层放 typescript/rust；仅框架放 vue/tauri；涉及多层时放最通用且最不易过期的层，其余层一句话引用。
- **删除**原 `project-gotchas.mdc`；更新 `self-evolve.mdc` 编排层指向 `knowledge/`；更新 `README.md` 两处结构说明。
- **安装机制无需改**：`bin/cli.js` 的 `cpSync recursive` 复制整个 `rules/`，自动包含 `knowledge/` 子目录。
- **编号延续**：拆分时保留原 F1–F13 / P1–P15 编号，新增专项用 `F/P-{lang}-N` 子编号，便于记忆与规则交叉引用。
