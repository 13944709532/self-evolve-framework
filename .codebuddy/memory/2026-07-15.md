## 新增 update-impeccable 命令（进化时自动同步上游）

- 实现 `src/cli/commands/update-impeccable.js`：
  - 读本地版本 `skills/impeccable/SKILL.md` frontmatter `version:`。
  - 查上游 `pbakaus/impeccable` 最新 release（GitHub API `releases/latest`），从 tag（如 `skill-v3.9.1`）解析版本。
  - 上游 skill 根路径为 `.agents/skills/impeccable/`（布局 SKILL.md + agents/ + reference/ + scripts/），用 `git/trees/<tag>?recursive=1` 取全部文件，再从 `raw.githubusercontent.com` 逐个下载到临时目录，最后原子替换 `skills/impeccable/`。
  - 健壮性：手动 `AbortController`+`setTimeout` 超时（Node 20.20.1 的 `AbortSignal.timeout` 会提前中止 fetch，不能用）；每个文件失败重试 3 次；**关键文件 SKILL.md 缺失则放弃替换保留原目录**；无网络仅告警跳过。
  - 已注册为 `self-evolve update-impeccable`（支持 `--force`/`--dry-run`），并在 `init.js` 维护者模式（cwd===PKG_ROOT 且包名 self-evolve-framework）下自动触发。普通用户安装不触发网络。
- 已文档同步：README（CLI 命令 + 自动更新说明）、AGENTS（命令中枢注释）、utils.js `showHelp()`。
- 实测把 bundled impeccable 从 3.8.0 升级到 **3.9.1**（99 文件）。

## 诊断：impeccable 调用路径与项目结构冲突（已修复）
- 用户报两个问题：(1)`fatal: unable to access 'https://github.com/13944709532/self-evolve-framework/': Recv failure: Connection was reset`；(2) 安装新版本后 impeccable 自身调用路径与项目路径冲突。
- **问题1 结论**：框架 CLI 无任何 git 远程调用；impeccable 脚本里唯一的 git 调用是 `scripts/context-signals.mjs`（本地 `git rev-parse/diff` 类）与 `lib/is-generated.mjs`（`git check-ignore`），均不访问远程。该 `Recv failure` 是用户自行 `git push/pull` 自己仓库时连接被重置（网络/代理），**非代码 bug**。修复建议：增大 `git config http.postBuffer 524288000`、改 `git config http.version HTTP/1.1`、或改用 SSH 远端 `git remote set-url origin git@github.com:13944709532/self-evolve-framework.git`。
- **问题2 根因**：上游 impeccable 按 `.agents` harness 编写，SKILL.md/reference/*.md/scripts 内对自身脚本的调用路径全是 `.agents/skills/impeccable/scripts/*.mjs`，但本框架把 skill 装到 `.codebuddy/skills/impeccable/`，目标项目 AI 按 SKILL.md 跑 `node .agents/...` 会找不到文件。另外 `scripts/pin.mjs` 的 `HARNESS_DIRS` 不含 `.codebuddy`，导致 pin 功能在本 harness 下失效。
- **修复**：在 `update-impeccable.js` 新增 `localizeSkill(dir)`，下载落到 `skills/impeccable/` 前执行：① 遍历所有文件把 `.agents/skills/impeccable` → `.codebuddy/skills/impeccable`；② 给 `scripts/pin.mjs` 的 `HARNESS_DIRS` 追加 `.codebuddy`。放在同步流程内，升级不会被上游覆盖冲掉。
- 实测：`update-impeccable --force` 同步 99 文件、本地化 8 个文件（SKILL.md + 5 个 reference 文档 critic/hooks/init/live/polish + hook-admin.mjs + pin.mjs）；旧 `.agents/skills/impeccable` 路径经正确通配符复核已清零。lints 0 错误。README 自动更新章节已补本地化说明。
- 注意（排查坑）：Windows `findstr /s /c:"..." 裸目录` 会把目录当文件名导致搜索无效，必须用 `skills\impeccable\*` 通配符；且 findstr 无匹配返回 exit 1，不能用 `&&` 串接验证命令。
- 已知限制：`.codebuddy` 的 hooks（设计检测器钩子）暂未单独加 provider 入口（hook-admin.mjs 的 `.agents` provider 内路径已被改写成 `.codebuddy`，但具体 `.codebuddy` hooks manifest 格式 CodeBuddy 是否支持未确认），不在本次"调用路径"范围内，留待后续。
