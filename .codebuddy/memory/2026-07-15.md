## 新增 update-impeccable 命令（进化时自动同步上游）
- 需求：进化（维护者在本仓库跑 self-evolve）时检查 impeccable 版本，有更新则同步项目中的 impeccable。
- 实现 `src/cli/commands/update-impeccable.js`：
  - 读本地版本 `skills/impeccable/SKILL.md` frontmatter `version:`。
  - 查上游 `pbakaus/impeccable` 最新 release（GitHub API `releases/latest`），从 tag（如 `skill-v3.9.1`）解析版本。
  - 上游 skill 根路径为 `.agents/skills/impeccable/`（布局 SKILL.md + agents/ + reference/ + scripts/），用 `git/trees/<tag>?recursive=1` 取全部文件，再从 `raw.githubusercontent.com` 逐个下载到临时目录，最后原子替换 `skills/impeccable/`。
  - 健壮性：手动 `AbortController`+`setTimeout` 超时（Node 20.20.1 的 `AbortSignal.timeout` 会提前中止 fetch，不能用）；每个文件失败重试 3 次；**关键文件 SKILL.md 缺失则放弃替换保留原目录**；无网络仅告警跳过。
  - 已注册为 `self-evolve update-impeccable`（支持 `--force`/`--dry-run`），并在 `init.js` 维护者模式（cwd===PKG_ROOT 且包名 self-evolve-framework）下自动触发。普通用户安装不触发网络。
- 已文档同步：README（CLI 命令 + 自动更新说明）、AGENTS（命令中枢注释）、utils.js `showHelp()`。
- 实测把 bundled impeccable 从 3.8.0 升级到 **3.9.1**（99 文件）。
- 布局修正：上游用 `scripts/`+`agents/` 平铺（无 `src/`），旧拷贝却是 `src/scripts`+`src/agents`，且 SKILL.md 引用 `scripts/...` 导致路径对不上——本次同步顺带修正了该 bug。
- 注意：`skills/` 被 gitignore，`git status` 为空，无法从 git 恢复，只能靠重下载修复。修复中途曾因 3 个文件瞬时 fetch 失败（含 SKILL.md）导致目录短暂损坏，后补下载这 3 个文件修复。
